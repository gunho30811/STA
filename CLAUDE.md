# CLAUDE.md

## DEVLOG.md 관리 (필수)

이 프로젝트는 `DEVLOG.md`에 날짜별 개발 일지를 남긴다.

- 코드 변경이나 이슈 해결로 커밋을 만들 때마다, 그 내용을 `DEVLOG.md`의 해당 날짜(`## YYYY-MM-DD`)
  섹션에 짧게 기록한다. 그날 섹션이 없으면 새로 만든다.
- 사용자가 "세션저장"이라고 말하면, 그 세션에서 한 작업을 `DEVLOG.md`에 정리해서 마무리하고 커밋한다.

## 협업 워크플로 (2인, GitHub Flow)

- `main`은 보호 브랜치. 직접 push 금지, PR을 통해서만 머지.
- 작업은 항상 `feature/짧은설명` 또는 `fix/짧은설명` 브랜치에서 시작 (작업 전 `main`을 최신화).
- 머지 후에는 브랜치 삭제.
- `DEVLOG.md` 등 같은 날 여러 사람이 건드릴 수 있는 파일은 본인이 쓴 섹션/줄에만 추가해서
  텍스트 충돌 시 손으로 풀기 쉽게 유지.
